# Rebol Lab

> 一个面向软件测试学习、AI 工具实践与求职作品集展示的个人开发者实验室。

Rebol Lab 不是一个普通的个人主页，而是一个持续迭代的个人 AI 工具实验室。  
它将学习路线、项目展示、软件测试工具、求职辅助工具和个人成长记录整合到一个统一的平台中。

当前版本定位为：

- 软件测试学习工作台
- AI 工具实验室
- 求职作品集展示平台
- 前端开发实践项目

## 核心功能

### 1. AI 测试用例生成器

根据用户输入的功能需求，自动生成测试分析内容。

支持生成：

- 功能点分析
- 正常测试用例
- 异常测试用例
- 边界值测试
- 安全测试点
- 接口测试建议
- 可能出现的 Bug

功能支持：

- 填入示例
- 生成测试用例
- 复制文本
- 复制 Markdown
- 清空输入
- 最近记录
- localStorage 本地保存

---

### 2. Bug 报告生成器

根据问题描述、复现步骤、实际结果、期望结果等信息，生成标准化 Bug 报告。

支持生成：

- Bug 标题
- 问题摘要
- 测试环境
- 严重程度
- 优先级
- 复现步骤
- 实际结果
- 期望结果
- 初步原因分析
- 建议排查方向
- 补充说明

适合用于：

- 软件测试学习
- 缺陷报告练习
- 面试项目展示
- 测试流程模拟

---

### 3. 项目包装助手

帮助用户将普通项目整理成适合简历、作品集和面试表达的项目描述。

支持生成：

- 项目一句话介绍
- 简历项目描述
- 项目亮点
- 技术难点
- 我的职责表达
- 面试讲解稿
- 面试官可能追问
- 项目优化方向

适合用于：

- 简历项目包装
- 面试前准备
- 作品集内容整理
- 项目复盘总结

## 技术栈

| 技术               | 说明                 |
| ------------------ | -------------------- |
| Vue 3              | 前端核心框架         |
| TypeScript         | 类型约束与工程化开发 |
| Vite               | 项目构建工具         |
| Vue Router         | 页面路由管理         |
| SCSS               | 样式组织与主题控制   |
| Element Plus Icons | 部分图标资源         |
| LocalStorage       | 本地历史记录存储     |

## 项目结构

```text
src
├─ assets              # 静态资源
├─ components          # 公共组件
├─ layouts             # 页面布局
│  └─ MainLayout.vue
├─ mock                # Mock 数据
├─ router              # 路由配置
│  └─ index.ts
├─ stores              # 状态管理预留
├─ styles              # 全局样式
│  └─ main.scss
├─ views
│  ├─ about            # 关于页面
│  ├─ home             # 首页
│  ├─ projects         # 项目展示页
│  ├─ roadmap          # 学习路线页
│  └─ tools            # 工具页面
│     ├─ ToolsView.vue
│     ├─ TestCaseGenerator.vue
│     ├─ BugReportGenerator.vue
│     └─ ProjectPackager.vue
├─ App.vue
└─ main.ts
```

## 本地运行

### 安装依赖

```bash
npm install
```

### 启动开发环境

```bash
npm run dev
```

启动后访问：

```text
http://localhost:5173
```

### 构建项目

```bash
npm run build
```

构建完成后会生成：

```text
dist/
```

## 项目特点

### 1. 从个人主页升级为工具型项目

相比普通个人主页，本项目不仅展示个人信息，还集成了多个真实可用的工具模块。

### 2. 聚焦软件测试方向

核心工具围绕软件测试学习与求职场景设计，包括测试用例生成、Bug 报告生成和项目包装。

### 3. 支持持续扩展

工具页面采用独立模块设计，后续可以继续添加 README 生成器、接口测试助手、学习复盘工具等功能。

### 4. 适合作为求职作品集

项目不仅展示前端开发能力，也能体现需求分析、测试思维、工具设计和项目表达能力。

## 当前说明

当前版本暂未接入真实 AI API。

现阶段的生成逻辑主要通过前端 Mock 规则完成，用于验证页面交互、工具流程和项目结构。  
后续可以接入 OpenAI、DeepSeek、小米 MiMo 或其他大模型 API，进一步提升生成内容质量。
